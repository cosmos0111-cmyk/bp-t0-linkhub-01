Agepo와 BPaaS 운영 시스템: 검색 자산을 관리하는 방식

Agepo 캠페인이 BPaaS의 Scouter, Architect, Ghost, Closer 흐름으로 검색 자산을 생성하고 운영하는 구조를 정리합니다.

Agepo는 단일 사이트가 아니라 운영 체계입니다

Agepo 캠페인은 개별 랜딩 페이지 하나를 만들고 끝내는 방식이 아니라, 키워드 발굴부터 사이트 생성, 포스팅, 링크 분배, 검색엔진 제출까지 이어지는 운영 체계를 기준으로 움직입니다.

BPaaS는 이 흐름을 Scouter, Architect, Ghost, Closer 단계로 나누어 관리합니다. 각 단계는 독립적인 역할을 가지지만, 최종 기준은 site-registry에 남는 배포 상태와 검색 접근성입니다.

판매 가능한 자산으로 남기는 설계

생성된 사이트는 임시 실험물이 아니라, 도메인, 로컬 원본, 배포 저장소, 색인 상태가 함께 추적되는 운영 자산입니다. 특정 키워드에서 노출이 시작되면 해당 사이트 자체가 매각 또는 대행 운영의 단위가 될 수 있습니다.

그래서 링크허브는 단순 링크 목록보다 더 엄격한 역할을 갖습니다. 어느 자산이 실제 도메인을 갖고 있고, 어떤 계층에서 어떤 캠페인에 속하는지를 사람이 읽을 수 있는 형태로 공개합니다.

링크허브가 맡는 위치

Link Hub는 Agepo 캠페인의 자산 안내판입니다. 루트 페이지는 대표 자산을 보여주고, 하위 문서는 BPaaS 운영 방식과 개별 인프라 역할을 설명합니다.

검색엔진이 따라갈 수 있도록 robots.txt, sitemap.xml, canonical URL을 함께 배포하며, 추가되는 커스텀 도메인은 registry와 배포 상태가 맞을 때만 허브 안에 노출합니다.
